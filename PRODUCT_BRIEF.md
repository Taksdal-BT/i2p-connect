# Product Brief

I2P Connect is a local-first communication platform concept for digital autonomy over I2P.

## Audience

Primary users:

- people new to I2P who need calm onboarding
- privacy-conscious communities
- educators and workshop facilitators
- builders who want reviewable, local-first communication tools

Secondary users:

- maintainers integrating I2P Connect into host apps
- reviewers checking security and release claims
- community contributors writing docs or onboarding content

## Product Promise

I2P Connect should help users understand local I2P readiness, create local identity foundations, verify contacts, and communicate through I2P-native flows after implementation and testing.

## Non-Goals

- Zoom clone behavior is rejected.
- Real-time video is not promised.
- Guaranteed anonymity is not promised.
- Full E2EE is not promised until implemented and tested.
- SASE and zero-trust compliance are not promised.
- Cloud storage of private messages, private destinations, private keys, router logs, or contact graphs is forbidden.

## MVP Shape

The first implementation should be small:

- local app shell
- local I2P status checks
- local identity model
- contact invite flow
- private message MVP
- experimental audio message prototype

Every feature should explain what is measured, what is local, what is experimental, and what is not implemented.
