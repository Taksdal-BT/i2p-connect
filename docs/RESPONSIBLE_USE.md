# Responsible Use

I2P Connect is intended for lawful, consensual, privacy-preserving communication and digital autonomy.

Privacy tools can protect dignity, association, research, and community resilience. They can also be misused. The product should make responsible use visible without fear marketing.

## User Expectations

Users should understand that:

- privacy tools reduce some risks but do not remove all risks
- guaranteed anonymity is not promised
- full E2EE is not promised until implemented and tested
- real-time video is not promised
- I2P Connect is not an emergency communication system
- local device compromise can undermine private communication
- unsafe contact verification can undermine private communication
- screenshots, exports, and logs can reveal sensitive information

## Product Requirements

Before risky actions, future UI should explain:

- what data stays local
- what data may leave the device
- what is being shared with a contact
- what is experimental
- what cannot be recovered if lost
- what diagnostics may reveal

## Prohibited Framing

Avoid:

- criminal symbolism
- fear-based adoption language
- conspiracy framing
- hacker-only aesthetics
- untraceable claims
- emergency-use claims
- mainstream video-conferencing expectations
- SASE or zero-trust compliance claims unless implemented and tested

## Community Posture

The project should invite careful review, responsible disclosure, and measured improvement. Security issues should be handled with minimal public exposure of sensitive details until a fix or mitigation path exists.
