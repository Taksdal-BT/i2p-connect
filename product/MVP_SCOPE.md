# MVP Scope

The MVP should prove a narrow local-first communication path without overstating privacy or transport properties.

## In Scope

- Local app shell.
- Measured local I2P readiness.
- Local identity model.
- Verified contact invite flow.
- Private message MVP after storage, transport, and cryptographic boundaries are implemented and tested.
- Experimental audio message prototype after messaging foundations exist.
- Beginner onboarding missions.
- Sanitized diagnostics.

## Out Of Scope

- Zoom clone behavior is rejected.
- Real-time video is not promised.
- Guaranteed anonymity is not promised.
- Full E2EE is not promised until implemented and tested.
- SASE and zero-trust compliance are not promised.
- Cloud storage of private keys, private destinations, private messages, router logs, or contact graphs is forbidden.

## MVP Acceptance Rules

Each MVP feature must state:

- what is real
- what is local
- what is measured
- what is experimental
- what is not implemented
- what validation was run
